## 工作规范

1. 使用 brainstorming skill 进行创意设计
2. 使用 sequential-thinking 进行结构化思考
3. 一次问一个问题，多选题优先
4. 以对话确认的内容为准，文档作为补充
5. 禁止使用 git 工具
6. 设计完成后更新 `玩法文档.md`
7. Memory MCP 只记录简洁要点，详细内容写在本地文档
