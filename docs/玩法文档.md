# ETN — Embody The Now 完整设计 & 技术实现文档（最终整合版）



本文档整合了游戏设计核心概念、所有系统设计以及 Godot 4.6/4.7 的具体实现方案，适用于单人独立开发者，目标 2026 年发布 Demo。



---



## 第一部分：游戏设计核心



### 1.1 项目概述

- **游戏名称**：ETN — Embody The Now（活在当下）

- **游戏类型**：剧情向 JRPG / 校园生活模拟 / 叙事驱动游戏

- **核心主题**："活在当下"，有些人注定只能陪你走一段路

- **改编来源**：作者真实高中经历，情感核心为转校生的到来与离去

- **目标平台**：PC（Steam/itch.io）

- **开发工具**：Godot 4.6/4.7（GDScript），AI辅助（阿里Coding Plan）

- **视觉风格**：伪 3D 场景 + 像素风 2D 角色，固定视角



### 1.2 核心设计理念

- **叙事模拟**：轨迹随机性低，改编真实经历，玩家扮演作者体验青春。

- **循环暗线**：作为背景设定，不显式表现，通过异常、记忆碎片、造梦主暗示。

- **结局哲学**：离别无法改变，但可以选择如何面对（多结局+彩蛋结局）。

- **无战斗 JRPG**：用探索、情绪管理、收集、好感度替代传统战斗，情感本身就是奖励。



### 1.3 核心系统概览

| 系统 | 简要说明 |

|------|----------|

| 存档/读档 | 仅大选项节点可读档，自动保存日常进度 |

| 情绪系统（G/D值） | 影响对话选项、心理状态，通过边框文字反馈 |

| 收集品 | 主线/支线获得，用于解锁区域，永久保留 |

| 好感度 | 进度条+阶段，试探机制获取对方好感 |

| 造梦主 | 每周/章节强制触发梦境指引 |

| 支线 | 时间窗口限制，过期无法开启，任务面板追踪 |

| 成绩 | 影响清北班淘汰、竞赛支线，但不强制限制自由 |

| 时间/日程 | 每日时段推进，NPC 按日程移动 |



### 1.4 故事与角色设计

#### 1.4.1 故事时间线

- **时间跨度**：高中三年（高一 ~ 高三）
- **核心事件**：转校生的到来与离去
- **叙事结构**：改编真实经历，轨迹随机性低

#### 1.4.2 角色设定

- **核心角色数量**：10-20 个
- **角色类型**：以同龄人（同年级同学）为主，无跨年级角色
- **差异化设计**：
  - 主线角色（2-3 个）：完整剧情线、深度好感系统
  - 支线角色（5-8 个）：独立支线任务、中等好感深度
  - 日常角色（10+ 个）：日常交互、简单好感

#### 1.4.3 关键角色：转校生

- **定位**：故事核心人物，转学来到主角所在班级
- **作用**：情感主线载体，玩家的选择将影响与转校生的关系走向
- **特殊性**：好感度培养不一定导向结局，需要与"合适的人"培养好感

### 1.5 存档与读档系统详解

#### 1.5.1 核心原则

- **玩家不能主动存档**：存档是系统行为
- **大选项自动存档**：每次选择大选项时系统自动保存
- **读档入口**：第一天结束后，在 [家] 中可以通过读档电脑回溯

#### 1.5.2 读档机制

- **读档入口位置**：家中（第一天结束后解锁）
- **读档限制**：原则上可无限读档，但会影响剧情走向和成就获取
- **通关后特权**：游戏通关后，可以不付出任何代价回到存档点

#### 1.5.3 设计理念

> 从设计角度来说，作者并不推崇读档操作。作者和文中的故事一样，希望大家能够收获一个在自己操作下的结局，而不是执着。

#### 1.5.4 防误触机制

- 点击选项先查看详细说明
- 点击确认后再执行选择

### 1.6 选项系统

#### 1.6.1 小选项

- **定义**：日常生活、对话、购物等日常选择，主题选择属于此类
- **特点**：
  - 不影响剧情走向
  - 轻微影响好感度（±1~5点）
  - 触发简单情绪效果（开心/难过等）
  - 可根据个人喜好选择
  - 非对话场景可以不做出选择
- **标记**：普通样式，无特殊标记
- **后果**：轻微，不会导致支线任务失败或重大剧情变化

#### 1.6.2 大选项

- **定义**：会影响剧情走向的关键选择，核心剧情环节
- **特点**：
  - 每次选择自动存档
  - 触发不同事件，可能导致不同结局
  - 需要仔细思考后选择
  - 无倒计时，玩家可无限思考
- **好感度影响**：±10~20点（较大幅度）
- **标记**：视觉+音效+动画组合提示，确保玩家能区分
- **后果类型**：
  - 剧情分支：不同选项→不同剧情线/演出顺序
  - 支线任务成败：包括NPC离去/关系破裂（主线会控制不发生）
  - 数值大幅变化：好感/精神状态剧烈波动
- **技术设计**：选项系统预留接口，选中特定选项后可执行代码/函数，保持可扩展性

#### 1.6.3 [Destiny] 命运

- **概念**：有些事情是一定会发生的
- **特点**：玩家的选择有时也没有那么强有力
- **作用**：强化"活在当下"主题，接受无法改变的事实
- **状态**：待完善具体机制

## 第二部分：Godot 技术实现方案



### 2.1 项目结构与全局设置



#### 2.1.1 文件夹组织

```



res://

├── assets/

│   ├── models/        # 3D模型（.glb）或 GridMap 材质

│   ├── sprites/       # 2D角色立绘/动画帧

│   ├── fonts/         # 像素字体

├── scenes/

│   ├── world/         # 校园场景（教学楼、食堂等）

│   ├── ui/            # UI场景（主菜单、背包、任务面板、对话弹窗）

│   ├── characters/    # 角色场景（Player, NPC）

│   └── systems/       # 系统场景（读档电脑、造梦主梦境）

├── scripts/

│   ├── global/        # 全局自动加载脚本

│   ├── systems/       # 各系统脚本

│   └── ui/            # UI脚本

├── data/              # 游戏数据（对话、任务、物品、NPC日程）

└── addons/            # 第三方插件



```



#### 2.1.2 全局自动加载（Autoload）

需在项目设置中添加：

- `GameState`：管理 G/D值、好感度字典、全局标志

- `SaveManager`：存档/读档

- `InventoryManager`：背包管理

- `TaskManager`：任务追踪

- `TimeSystem`：日期/时间推进，NPC 日程调度

- `DialogSystem`：对话控制（可封装插件或自定义）



### 2.2 伪3D场景搭建（无需 Blender）



#### 2.2.1 使用 GridMap 构建校园

- **创建 MeshLibrary**：

  - 新建 `MeshLibrary` 资源。

  - 添加基本几何体（`BoxMesh`、`CylinderMesh` 等），设置材质为像素风格贴图（可用 16x16 重复纹理）。

  - 导出为 `.tres` 文件。

- **搭建场景**：

  - 添加 `GridMap` 节点，指定 `mesh_library`。

  - 设置 `cell_size` 为网格单元大小（如 2.0）。

  - 手动放置方块组合成教学楼、走廊、食堂等。

- **复杂形状**：使用 `CSG` 节点（`CSGBox3D`、`CSGCombiner3D`）组合特殊结构。

- **碰撞**：`GridMap` 可自动生成碰撞，或在需要的地方手动添加 `StaticBody3D`。



#### 2.2.2 像素化视觉效果

- 在 `WorldEnvironment` 中添加 `Environment`，设置 `tonemap` 为 `Filmic`，加轻微晕影。

- 添加自定义着色器（`shaders/pixelize.gdshader`）模拟像素化，或使用 Godot 的 `SCREEN_TEXTURE` 降低分辨率。



### 2.3 角色系统



#### 2.3.1 玩家角色（Player）

**节点结构**：

```



CharacterBody3D (Player)

├── CollisionShape3D (胶囊形)

├── Sprite3D (2D角色，billboard 模式)

│   └── AnimationPlayer (控制帧动画)

├── CameraPivot (Node3D)

│   └── Camera3D (固定视角，如斜45度)

└── InteractionArea (Area3D，用于检测交互)



```

**Player.gd**（简化）：

```gdscript

extends CharacterBody3D



@export var speed = 5.0

@export var interaction_distance = 2.0



func _physics_process(delta):

    var input_dir = Input.get_vector("left", "right", "forward", "back")

    var direction = (transform.basis * Vector3(input_dir.x, 0, input_dir.y)).normalized()

    velocity.x = direction.x * speed

    velocity.z = direction.z * speed

    move_and_slide()



func _input(event):

    if event.is_action_pressed("interact"):

        interact()



func interact():

    var space = get_world_3d().direct_space_state

    var query = PhysicsRayQueryParameters3D.create(global_position, global_position - global_transform.basis.z * interaction_distance)

    query.collision_mask = 2  # 交互层

    var result = space.intersect_ray(query)

    if result and result.collider.has_method("on_interact"):

        result.collider.on_interact()

```





2.3.2 NPC 角色（带日程、自动导航、动画）





节点结构：





```



CharacterBody3D (NPC)

├── CollisionShape3D

├── Sprite3D + AnimationPlayer

├── NavigationAgent3D

└── InteractionArea (触发对话)

```





NPC.gd：





```gdscript

extends CharacterBody3D



@export var npc_id: String

@export var schedule_data: Dictionary  # 从 JSON/Resource 加载

@export var speed = 3.0



@onready var nav_agent = $NavigationAgent3D

@onready var anim_player = $AnimationPlayer



var current_target: Vector3

var current_animation = "idle"



func _ready():

    # 连接时间系统信号

    TimeSystem.time_advanced.connect(_on_time_advanced)

    # 初始化第一次位置

    update_schedule()



func _physics_process(delta):

    if nav_agent.is_navigation_finished():

        if anim_player.current_animation != current_animation:

            anim_player.play(current_animation)

        return

    var next = nav_agent.get_next_path_position()

    var direction = (next - global_position).normalized()

    velocity = direction * speed

    move_and_slide()

    if anim_player.current_animation != "walk":

        anim_player.play("walk")



func _on_time_advanced(day: int, period: String):

    update_schedule(day, period)



func update_schedule(day: int, period: String):

    # 根据日程数据查找目标地点和动作

    var entry = find_schedule_entry(day, period)

    if entry:

        var target_pos = get_location_from_id(entry.location)

        move_to(target_pos)

        current_animation = entry.animation



func move_to(pos: Vector3):

    current_target = pos

    nav_agent.target_position = pos



func on_interact():

    # 触发对话

    DialogSystem.start_dialogue(npc_id)

```

```json
  { "day": 1, "period": "class", "location": "classroom_302", "animation": "sit" }

]

```





· 导航网格：场景中添加 NavigationRegion3D，子节点包含 GridMap，烘焙导航。

2.4.1 数据格式选择






```gdscript

# dialogue_choice.gd

@export var next_id: String

@export var condition: String

@export var effects: Dictionary
```

· 加载方式：load("res://data/dialogues/chapter1.tres")





2.4.2 对话系统管理器（可复用用户弹窗）





· 用户已有弹窗系统，需适配新数据源并修改 UI 风格为像素风：

  · 去除液态玻璃效果（高斯模糊、半透明）。

  · 使用纯色背景、硬边框、像素字体（如"Press Start 2P"）。

  · 布局：底部文本框，左侧说话人，选项列表。

· 核心逻辑：





```gdscript

# DialogSystem.gd (autoload)

extends Node



var current_dialogue: Resource

var current_node_id: String



func start_dialogue(npc_id: String, chapter: String = "chapter1"):

    current_dialogue = load("res://data/dialogues/" + chapter + ".tres")

    current_node_id = "start"

    show_dialogue_balloon()



func show_dialogue_balloon():

    var node = get_node_by_id(current_node_id)

    if not node:

        return

    # 检查条件

    if node.condition and not evaluate_condition(node.condition):

        current_node_id = node.next_id  # 跳过

        show_dialogue_balloon()

        return

    # 显示文本和选项

    var balloon = preload("res://scenes/ui/DialogueBalloon.tscn").instantiate()

    balloon.set_data(node)

    add_child(balloon)

    balloon.choice_selected.connect(_on_choice_selected)



func _on_choice_selected(choice: DialogueChoice):

    apply_effects(choice.effects)

    current_node_id = choice.next_id

    show_dialogue_balloon()

```

### 2.4.4 事件系统详解

游戏设置多种影响游戏结局走向和参数的事件。一切涉及心理活动的事件都将被定义为[事件]。

#### 事件类型总览

| 事件类型 | 触发方式 | 主要效果 |
|----------|----------|----------|
| [快乐] | 选择触发/偶遇 | +G值，改好感，双向 |
| [难过] | 选择触发/偶遇 | +D值，改好感，双向 |
| [被关注] | 被动触发(偶遇) | 反映对方好感，+G -D |
| [关注] | 主动触发 | 改好感，影响GD值 |
| [试探] | 主动触发 | 获取对方好感度信息 |
| [???] | 选择触发 | 未知结果，可能转换为其他类型 |
| [中立] | 默认 | 不影响主线和好感 |

#### 事件详细说明

**[快乐]**
- 一般为选择触发或偶遇
- 可能提升G值
- 触发精神状态
- 能够改变好感度（原则上为双向）
- 是绝大多数事件的定义类型

**[难过]**
- 一般为选择触发或偶遇
- 可能提升D值
- 触发精神状态
- 能够改变好感度（原则上为双向）
- 是绝大多数事件的定义类型

**[被关注]**
- 一般为偶遇即被动触发
- 可以反映对象对我们的[对我的好感]
- 会体现在好感度UI中
- [对我的好感]越高越容易触发
- 能够大大提升[对Ta的好感]
- 触发某种精神状态，提升G值减少D值

**[关注]**
- 主动触发
- 发挥主观能动性
- 发自内心或故意关注对象
- 以达到改变好感度为目的
- 可能触发其他事件类型
- 可能会影响GD值

**[试探]**
- 用于获取对象[对我的好感]
- 使用不恰当会影响GD值
- 更会影响好感
- 成功时提示冰川小未的心理状态和好感情况

**[???]**
- 现阶段不知道结果
- 选择后事件性质可能转换为上述其他选项
- 增加不确定性

**[中立]**
- 对主线和好感并无太大影响
- 可根据自己对于角色的理解和喜好选择
- 默认该选项不做标记

### 2.4.5 精神状态系统详解

精神状态通过选项直接或间接触发，对游戏体验产生深远影响。

#### 状态总览

| 状态 | 持续性 | 主要效果 | 风险 |
|------|--------|----------|------|
| [Overjoyed] | 当前事件段 | 免疫负面效果 | 结束后易触发内耗 |
| [空虚] | 持续 | 不增G，选项减少 | 可转为难过 |
| [内耗] | 持续 | 自我怀疑，好感度系统不稳定 | 可转为抑郁 |
| [抑郁] | 持续 | 好感度系统崩溃，关系风险 | 可能导致BE |
| [神威] | 隐藏 | 待定 | - |

#### 状态详细说明

**[Overjoyed] 心动**
- 仅当前事件段有效，游戏中会标明
- 此状态下触发的难过事件或遭到拒绝也不会对好感产生负面影响
- 不会触发负面精神状态
- 有助于提升G值和[对Ta的好感]
- 注意：结束后不要触发[内耗]

**[空虚]**
- 触发方式：接受和自己关系不是很好的人的邀请；戒断反应
- 该状态下[快乐]事件不会增加G
- 可触发选项减少
- 选项中的事件更多是中立和[难过]

**[内耗]**
- 进入自我怀疑和焦虑状态
- 甚至会怀疑好感度系统（此状态下请勿完全相信好感度系统，会明确标注"好感度不准确"）
- 降低[对Ta的好感]，降低G值，提升D值
- 如果多次触发可能会进入[抑郁]

**[抑郁]**
- 在GD较高时（较高的GD值都有触发风险）遭受打击触发
- 在[Overjoyed]结束后注意不要触发[内耗]
- 好感度系统崩溃，此时只能查看[对ta的好感]
- 该状态下可能会对关系造成严重影响（可能导致BE，速通该章节）
- 注意：避免前往有风险的地方，不要让主角死亡（死亡会达成某种结局）

**[神威]**
- 隐藏触发
- 具体效果待定

### 2.4.6 好感度系统详解

#### 双向好感度

- **[对TA的好感度]**：玩家可实时查看，表示玩家对角色的好感
- **[对我的好感]**：角色对玩家的好感，需要通过[试探]事件获取

#### 核心机制

- 获取对方好感需要通过[试探]事件
- 不恰当的[试探]可能会损失好感
- 想要获取游戏胜利，必须得知对方对我们的好感度
- 和某一些角色确实可以培养非常高的好感度，但不一定能够引导走向结局
- 需要和"合适的人"培养好感

#### 好感度UI

- 进度条显示
- 阶段标记（陌生人→认识→朋友→好友→挚友等）
- 抑郁状态下只能查看[对ta的好感]

### 2.4.7 G值与D值系统详解

#### G值（Glad）

- 每次[快乐]事件可能增加G值
- 较好的G值有助于避免[内耗]并隐藏D值
- 越高G值下发生的[心动]事件越能减少D值
- 不同的GD值可能导致出现不同的选项

**G值与事件影响的关系：**
- 当前G值越高，越容易因事件或选择进入[内耗]
- 当前G值越高，[对Ta的好感度]越高，[难过]事件和[内耗]所造成的[对Ta好感度]和[D值]影响越严重

#### D值（Depression）

- 每次[难过]事件可能增加D值
- 在较高D值下更容易触发[内耗][抑郁][心动]
- D值过高会快速扭转心理状态

**D值与事件影响的关系：**
- 当前D值越高，[对Ta的好感度]越高，[快乐]事件所造成的[对Ta好感度]和[G值]影响越大

#### GD值交互

| 状态 | G值影响 | D值影响 |
|------|---------|---------|
| [快乐]事件 | +G | 可能减少D |
| [难过]事件 | 可能减少G | +D |
| 高G状态 | [心动]效果好 | 隐藏D值 |
| 高D状态 | 更易触发负面状态 | 更易抑郁 |

### 2.11 时间与日程系统

#### 2.11.1 时段划分

**周内（5时段）**

| 时段 | 活动机会 | 说明 |
|------|----------|------|
| 早读 | ❌ 无 | 强制，剧情推进 |
| 上课 | ✅ 大课间可活动 | 强制上课，但大课间有活动机会 |
| 午休 | ✅ 有行动点 | 用餐 + 自由活动 |
| 放学后 | ✅ 有行动点 | 实为晚饭时间，主要自由时段 |
| 夜间 | ✅ 有行动点 | 晚自习可选，行动点耗尽后必须就寝 |

**周末（3时段）**

| 时段 | 活动机会 | 说明 |
|------|----------|------|
| 上午 | ✅ 有行动点 | 自由活动 |
| 下午 | ✅ 有行动点 | 自由活动 |
| 夜间 | ✅ 有行动点 | 自由活动 |

**设计理念**
- 周内上课强制，但课间/放学后可活动
- 周末自由度更高，行动点更宽松
- 时间推进由剧情驱动，非固定日历循环

#### 2.11.2 行动点系统

**行动点分配（周内）**

| 时段 | 行动点 | 可用活动 |
|------|--------|----------|
| 上课（大课间）| 1-2点 | 自由活动/学习/社交 |
| 午休 | 1点 | 用餐/自由活动 |
| 放学后 | 1-2点 | 自由活动/社交 |
| 夜间 | 1-2点 | 晚自习/回家/寝室活动 |

**行动点分配（周末）**

| 时段 | 行动点 | 说明 |
|------|--------|------|
| 上午 | 2-3点 | 更宽松 |
| 下午 | 2-3点 | 更宽松 |
| 夜间 | 1-2点 | 与周内相同 |

**设计目的**
- 限制玩家活动，模拟现实时间限制
- 让玩家必须精心选择焦点
- 周末给予更多自由度

#### 2.11.3 用餐系统

**用餐方式与行动点消耗**

| 吃饭方式 | 消耗行动点 | 效果 |
|---------|-----------|------|
| 就近食堂 | 0点 | 仅解决用餐需求 |
| 校内其他地点（面包房/奶茶店）| 1点 | +好感（和朋友一起）|
| 商场/校外 | 2点 | 增进好感更多，可能触发特殊剧情 |

**设计说明**
- 早饭/午饭/晚饭都需要用餐
- 就近吃饭不消耗行动点，但无社交收益
- 去特色地点吃饭消耗行动点，但有社交收益

#### 2.11.4 夜间流程详解

**流程图**

```
进入夜间时段
    │
    ├─ 有行动点？
    │   ├─ 是 → 选择活动：
    │   │   ├─ 留校晚自习（提升成绩）
    │   │   ├─ 校内自由活动
    │   │   ├─ 主动回寝室 → 保留1点 → 寝室活动
    │   │   └─ 主动回家 → 保留1点 → 出去玩/在家活动
    │   │
    │   └─ 否（行动点=0）→ 必须就寝：
    │       ├─ 回寝室 → 直接睡觉
    │       └─ 回家 → 可进行系统操作，然后睡觉
    │
    └─ 选择睡觉 → 结束当天，推进到第二天
```

**回家后的活动分类**

| 活动类型 | 消耗行动点 | 效果 |
|---------|-----------|------|
| 看电视/玩游戏 | ❌ 不消耗 | 成绩下降 |
| 熬夜学习 | ❌ 不消耗 | 有概率提高成绩（影响第二天上课）|
| 给同学打电话 | ✅ 消耗1点 | 后期功能，社交 |
| 玩电脑/读档 | ❌ 不消耗 | 系统操作 |

**关键规则**
- 行动点耗尽后：不能进行消耗行动点的玩法活动，但可进行系统操作
- 主动回寝/回家：若有剩余行动点，保留1点可供活动
- 在家的一般活动（看电视、玩游戏）不消耗行动点，但影响成绩

#### 2.11.5 走读与住校

**当前设计**
- 走读/住校在玩法上无本质区别
- 回家后的限制主要在行动点系统
- 周内不能出学校玩，但可以选择回家

**回家功能**
- 电脑读档入口（核心功能）
- 未来扩展：玩游戏、去商场购物（需朋友邀请）

#### 2.11.6 周循环与特殊事件

**周循环**
- 自然周为单位
- 每周可能有特殊事件
- 造梦主梦境定期触发（周末）

**月考/学期节点**
- 月考影响清北班淘汰
- 成绩系统与学习次数关联
- 具体时间节点根据剧情需要设定

```gdscript
enum Period { MORNING, CLASS, LUNCH, AFTER_SCHOOL, NIGHT }

        if current_period > WeekendPeriod.NIGHT:
            current_period = WeekendPeriod.MORNING
            advance_day()
    else:
        current_period += 1
        if current_period > Period.NIGHT:
            current_period = Period.MORNING
            advance_day()
    
    time_advanced.emit(day, current_period, is_weekend)

func advance_day():
    day += 1
    # 根据剧情需要判断是否周末
    check_weekend()
    day_ended.emit()

func check_weekend():
    # 根据剧情时间表判断是否周末
    pass

func get_action_points() -> int:
    if is_weekend:
        match current_period:
            WeekendPeriod.MORNING, WeekendPeriod.AFTERNOON:
                return 3
            WeekendPeriod.NIGHT:
                return 2
    else:
        match current_period:
            Period.CLASS:
                return 2  # 大课间
            Period.LUNCH, Period.AFTER_SCHOOL:
                return 2

                return 2
            _:
```
